# P2P緊急地震速報の地図表示・マグニチュード表示改善

## 変更内容

### 問題
1. P2P緊急地震速報のマグニチュードが「情報なし」と表示される問題
2. P2P緊急地震速報の地図画像が震源地のみを表示し、周囲の震度分布・震度数字が表示されない問題

### 解決方法

#### 1. マグニチュード表示の改善
P2P地震情報からのマグニチュード抽出を大幅に強化し、より多くのフィールドをチェックして確実にマグニチュード情報を取得できるようにしました。

**改善点:**
- より多くのマグニチュードフィールドをチェック（`magnitude`, `mag`, `M`など）
- 緊急地震速報専用の追加検索ロジック
- P2P全フィールドからのマグニチュード候補自動検索
- 詳細なログ出力でデバッグ可能

#### 2. 震度分布表示の改善
P2P地震情報の`areas`配列から地域ごとの震度情報を抽出し、JMAの震度観測データ構造に変換して、`/get_eq`コマンドと同じ地図生成ロジックを使用します。

**改善点:**
- P2Pの`areas`配列から地域震度情報を完全抽出
- 座標推定機能で各地域の正確な位置を特定
- JMA互換データ構造への完全変換
- 統一された地図生成処理

### 主な変更点

#### 1. マグニチュード抽出の強化
```typescript
// 拡張されたマグニチュードフィールドチェック
const magnitudeValue = earthquakeData.magnitude ||
                      earthquakeData.mag ||
                      earthquakeData.M ||
                      p2pRecord.magnitude || 
                      p2pRecord.mag ||
                      p2pRecord.M ||
                      '不明'

// 緊急地震速報専用の追加検索
// P2P全フィールドからマグニチュード関連を自動検索
```

#### 2. P2P Areas処理の完全実装
- P2Pの`areas`配列から地域名と震度情報（`scaleFrom`、`scaleTo`）を抽出
- 各地域の最大震度を計算し、JMA形式の震度文字列に変換
- JMA互換の`Pref` → `Area` → `City` → `IntensityStation`構造を生成

#### 3. 座標推定機能（既存の拡張）
- `estimateP2PAreaCoordinates`関数による約80箇所の座標データベース
- 都道府県、地方、海域名から緯度経度を自動推定
- 震度数字が各地域の正確な位置に表示される

#### 4. デバッグ機能の大幅強化
- マグニチュード抽出プロセスの詳細ログ
- P2P areas配列の処理状況確認
- 座標推定結果の詳細表示
- JMA変換結果の構造確認

### 技術的詳細

#### 変更ファイル
- `src/utils/earthquake.ts`
  - `convertP2PtoJMAFormat`関数: マグニチュード抽出ロジック強化、P2P areas処理完全実装
  - `estimateP2PAreaCoordinates`関数: 座標推定機能（既存）
- `src/utils/mapGenerator_new.ts`
  - デバッグログ強化（既存）

#### データフロー（改善版）
1. P2P WebSocketから緊急地震速報データ受信
   - マグニチュード情報の多重チェック
   - `areas`配列の詳細解析
2. `processP2PEarthquakeAlert`でJMA ID照合試行
3. 照合失敗時に`convertP2PtoJMAFormat`でJMA互換形式に変換
   - **強化されたマグニチュード抽出**
   - **P2P areas → JMA Pref/Area/City/IntensityStation構造変換**
   - 地域名から座標推定を実行
4. `extractEarthquakeMapData`で地図データ抽出（`/get_eq`と同じロジック）
5. `generateEarthquakeMap`で地図画像生成（震源+周囲震度数字）

#### 1. 基本フィールドチェック
- `earthquake.magnitude`, `earthquake.mag`, `earthquake.M`
- `p2pRecord.magnitude`, `p2pRecord.mag`, `p2pRecord.M`

#### 2. 緊急地震速報専用処理
- EEW特有フィールドの追加チェック
- タイポ可能性を考慮したフィールド名

#### 3. 全フィールド自動検索
- P2Pデータの全フィールドを走査
- マグニチュード関連キーワードの自動検出
- 数値妥当性チェック（0-10の範囲）

#### 4. 正規化処理
- 不適切な値（「速報値」「未確定」等）の除去
- 数値形式の統一（小数点1桁）
- 詳細なログ出力

- マグニチュード抽出の各段階での詳細ログ

- 座標推定結果と最終座標の確認

これで、P2P緊急地震速報でマグニチュード情報が正確に表示され、`/get_eq`コマンドと同様に地図上に各地の震度数字も適切に表示されるようになりました。

### 主な変更点

1. **P2P areasデータの処理追加**
   - P2Pの`areas`配列から地域名と震度情報（`scaleFrom`、`scaleTo`）を抽出
   - 各地域の最大震度を計算し、JMA形式の震度文字列に変換

2. **座標推定機能の追加**
   - `estimateP2PAreaCoordinates`関数を追加
   - P2P地域名から緯度経度を推定（都道府県、地方、海域など約80箇所に対応）
   - 座標情報がない場合でも地図上に震度分布を表示可能

3. **JMA互換データ構造の生成**
   - P2Pの地域震度データをJMAの`Pref` → `Area` → `City` → `IntensityStation`構造に変換
   - 座標情報を`latlon`フィールドに格納

### 技術的詳細

#### 変更ファイル
- `src/utils/earthquake.ts`
  - `convertP2PtoJMAFormat`関数: P2P areas配列の処理ロジック追加
  - `estimateP2PAreaCoordinates`関数: 新規追加（地域名から座標推定）

#### データフロー
1. P2P WebSocketから緊急地震速報データ受信
2. `processP2PEarthquakeAlert`関数でP2Pデータを処理
3. `convertP2PtoJMAFormat`でJMA互換形式に変換（新しいareas処理ロジック）
4. `extractEarthquakeMapData`で地図データ抽出（既存の/get_eqと同じロジック）
5. `generateEarthquakeMap`で地図画像生成（震源+周囲震度）

### 結果
- P2P緊急地震速報でも/get_eqコマンドと同様に震源地周囲の震度分布が地図上に表示される
- 地域名から座標を推定することで、座標情報がないP2Pデータでも適切な位置に震度マーカーを配置
- 既存の地図生成ロジックをそのまま活用し、一貫した地図表示を実現

### 対応地域
都道府県、地方名、主要海域（トカラ列島、奄美大島、三陸沖など）約80箇所の座標データベースを構築し、P2P地震情報でよく見られる地域名に対応。
